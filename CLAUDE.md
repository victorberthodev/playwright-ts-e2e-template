## What this is

Production-grade Playwright + TypeScript E2E template. Use it to bootstrap
a new automation project — clone, change the URLs in `.env.dev`, start
writing specs in `tests/`.

- UI under test: [SauceDemo](https://www.saucedemo.com)
- API under test: [JSONPlaceholder](https://jsonplaceholder.typicode.com)
- Reporting: Allure + Playwright HTML
- CI/CD: GitHub Actions (matrix on chromium/firefox/webkit)

## Conventions

- **Imports** — specs import `test` and `expect` from `@/fixtures`, never
  directly from `@playwright/test`. Path alias `@/*` → `src/*`.
- **Page Objects** — typed `Locator` getters, no assertions inside the
  POM (enforced by ESLint `no-restricted-syntax`).
- **Selectors** — `getByRole` → `getByLabel` → `getByTestId` →
  `getByText`. No XPath, no generated CSS classes.
- **No `waitForTimeout`** — use web-first matchers (`toHaveText`,
  `toBeVisible`) or `expect.poll`. ESLint blocks `waitForTimeout`.
- **Tags** — inline metadata, not in test name:
  `test('...', { tag: ['@smoke', '@auth'] }, async () => { ... })`.
- **Commits** — Conventional Commits, types limited to `feat | fix |
test | chore | docs | refactor | ci | perf | build | style | revert`.
  See [CONTRIBUTING.md](./CONTRIBUTING.md).
- **TypeScript** — strict + `noUncheckedIndexedAccess` +
  `exactOptionalPropertyTypes`. No `any` without a justified ESLint
  disable.

## Folder layout

```
src/
├── api/         # clients, schemas (zod), DTO types
├── pages/       # Page Objects
├── fixtures/    # Playwright fixtures + merged `test` export
├── data/        # constants + faker-backed builders
├── utils/       # logger, env loader
└── config/      # environments, constants, allure config
tests/
├── *.setup.ts   # setup project (storageState)
├── ui/          # UI specs
├── api/         # API specs
└── hybrid/      # API + UI hybrid specs
playwright/.auth/  # generated storageState (gitignored)
```

## Environment

`src/utils/env.ts` loads `.env` then `.env.${TEST_ENV}` (defaults to
`dev`) and validates with zod. Cross-platform — no `dotenv-cli` in
scripts:

- bash: `TEST_ENV=stg pnpm test`
- PowerShell: `$env:TEST_ENV='stg'; pnpm test`

Copy the `.env.*.example` files to `.env.*` and adjust per environment.

## Adding things

- **Page Object** — extend `BasePage`, expose typed `Locator` getters,
  add business actions, override `goto()` if reachable by URL.
- **Fixture** — define in `src/fixtures/<name>.fixture.ts`, then
  `mergeTests` it into `src/fixtures/index.ts`.
- **API client** — schema in `src/api/schemas/`, types via `z.infer` in
  `src/api/types/`, client in `src/api/clients/`.
- **Test** — file under `tests/{ui,api,hybrid}/<feature>/`, import from
  `@/fixtures`, tag inline, add Allure annotations
  (feature/story/severity).

See [CONTRIBUTING.md](./CONTRIBUTING.md) for the full workflow.

## Constitution (non-negotiable)

1. **Determinism > speed** — no `waitForTimeout`, no flaky selectors,
   retries only in CI.
2. **Test isolation** — each test owns its state; storage state
   generated by the `setup` project, never committed.
3. **POM stays thin** — composition > inheritance, no assertions inside
   page objects.
4. **Layer separation** — `tests/` (intent) vs `src/pages` (UI) vs
   `src/api` (HTTP) vs `src/fixtures` (composition).
5. **Strict TypeScript** — see tsconfig.
6. **Observability by default** — trace + screenshot + video on failure,
   Allure annotations, `step()` over `console.log`.
7. **Conventional commits + branch prefixes** — `feat/`, `fix/`,
   `chore/`, `docs/` — enforced by Husky + commitlint.
